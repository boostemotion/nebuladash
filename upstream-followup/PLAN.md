# 上游跟进计划

## 目标

在不破坏当前 NebulaDash 可用性的前提下，有节奏地吸收 `zashboard` 上游有价值更新，减少未来维护成本。

## 当前判断

当前不适合直接全量 rebase 到上游 `3.11.x`，原因有三点：

1. 本地已有较多二改，特别是代理页、规则搜索、OpenClash 适配、路由入口和设置项
2. 你当前最敏感的问题仍是“代理页加载与可用性”，不是主题外观
3. 上游 `3.x` 包含较多 UI/交互级重构，直接吞并回归风险高

## 总策略

- 路线：`功能分层回收`，不是 `一次性追平`
- 方法：优先 cherry-pick / 手动移植，最后再评估大版本对齐
- 节奏：按 `P0 -> P1 -> P2 -> P3`

## P0 稳定性优先

目标：先把当前最影响使用的体验稳住。

任务：

- 保持当前 OpenClash 单 provider 方案稳定
- 继续压低代理页“加载中像空数据”的误导感
- 保留当前 NebulaDash 针对慢 `/providers/proxies` 的本地优化
- 建立一份最小回归清单：
  - 代理页可打开
  - 策略组能显示
  - 节点组能显示
  - provider 页可接受慢，但不能拖死主页
  - 设置页可正常切换主题和后端

完成标准：

- 路由器上日常使用无明显“打开即 0 组”的误判
- `proxies` 页主要功能可用

## P1 值得回收的上游功能

这批功能价值高、风险相对可控，建议优先吸收。

### 1. 代理文件夹管理

上游版本：
- `3.7.0`

价值：
- 适合节点和策略组较多的场景
- 有助于进一步解决代理页过长、信息密度过高的问题

风险：
- 会碰代理页渲染逻辑，需要与你当前的分组聚合逻辑一起评估

建议：
- 单独分支验证，不直接进主线

### 2. 节点搜索模式 / 搜索增强

上游版本：
- `3.6.0`
- `3.7.0`

价值：
- 你当前节点多、来源多，搜索体验提升很有价值

风险：
- 中低，主要集中在代理页搜索逻辑

建议：
- 优先纳入

### 3. provider 搜索和 provider 组件优化

上游版本：
- `3.5.1`
- `3.2.0`

价值：
- 即使你现在走单 provider，也仍然能受益于更稳的 provider 视图逻辑

风险：
- 中，需要和你当前的 provider 缓存逻辑核对

### 4. 规则卡片动画与交互优化

上游版本：
- `3.9.0`

价值：
- 纯体验增强

风险：
- 低

建议：
- 可穿插吸收

## P3 暂缓项

这些不是不能做，而是不适合现在做。

### 1. 全量追平 zashboard@3 视觉体系

原因：
- 回归面太大
- 会和你当前二改的视觉、布局与 OpenClash 场景相互冲突

### 2. 直接 rebase 到 upstream/main

原因：
- 当前本地改动面较广
- 容易让代理页、设置页、路由器适配一起炸

### 3. goroutines tracking

原因：
- 上游该功能本质上依赖 `sing-box native API` 的 `subscribeStatus` 流
- 当前 NebulaDash 主要跑的是 `OpenClash / Mihomo` WebSocket 接口，不具备同等数据来源
- 就算移植 UI，也大概率只有空值或必须额外引入整条 sing-box native 适配链

结论：
- 从当前主计划中删除，不作为独立功能推进
- 将来只有在“明确引入 sing-box native API 支持”时再一并评估

### 4. 快捷键体系

原因：
- 用户已明确不需要
- 对当前路由器/OpenClash 使用场景提升有限

结论：
- 从当前计划中删除

## 建议落地节奏

### 阶段 A：稳线整理

- 固化当前可用版本
- 完成回归清单
- 保证每次升级前后都能验证

### 阶段 B：小步吸收

- 搜索增强
- provider 视图增强
- 规则卡片交互

### 阶段 C：代理页专项升级

- 评估并接入“代理文件夹管理”
- 与你现在的“分组/节点来源/单 provider”方案融合

### 阶段 D：长期对齐

- 视情况评估 sing-box native 支持
- 再决定是否做大版本 UI 对齐

## 建议的分支策略

- `main`
  当前稳定可用线
- `feat/upstream-search-sync`
  搜索相关上游跟进
- `feat/upstream-provider-sync`
  provider 相关上游跟进
- `feat/upstream-folder-mode`
  代理文件夹模式验证
- `spike/upstream-3x-rebase`
  大版本对齐试验线，不进入主线

## 每次上游跟进的验收项

- `pnpm type-check` 通过
- `pnpm build` 通过
- 代理页能打开
- 策略组数量正确
- 节点组数量正确
- provider 页不会阻塞主页
- 规则页搜索正常
- 设置页可正常保存关键配置
